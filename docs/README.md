# Launch / store assets

Visual assets for the README and store listings live here.

## Store screenshots (1280×800, ready to upload)

Generated by `make_screenshots.py` — they reproduce a GitHub PR page in dark
theme with the **real** MergeChain block (CSS read straight from
`src/content/dom.ts`), so they can't drift from the shipped UI.

| File | Shows |
|---|---|
| `screenshots/01-blocked.png` | Blocked PR — red merge button, "Blocked by 2 PRs" |
| `screenshots/02-unblocks.png` | All clear — "All dependencies are merged", merge enabled |
| `screenshots/03-declare.png` | Declaring a dependency (autocomplete + direction pill) |
| `screenshots/04-chain.png` | Transitive / cross-repo chain + reverse "depend on this" |
| `screenshots/05-private.png` | No backend / token-local message + placement popup |

Regenerate:

```bash
python3 docs/make_screenshots.py          # headless Chrome (CI / most machines)
# On a machine where headless Chrome hangs, emit HTML and render with gstack browse:
MC_HTML_ONLY=1 python3 docs/make_screenshots.py
B=~/.claude/skills/gstack/browse/dist/browse
$B viewport 1280x800 --scale 2
for f in docs/screenshots/*.html; do p=${f%.html}.png; $B goto "file://$PWD/$f"; $B screenshot "$PWD/$p"; sips -z 800 1280 "$p"; done
rm docs/screenshots/*.html
```

## Tips

- Use a clean throwaway repo (e.g. `oleg-koval/pr-deps-e2e`) with obvious PR
  titles ("PR A: base feature", "PR B: depends on A").
- Dark theme reads best against the extension's Primer-token styling.
- For the GIF, keep it under ~3 MB so it inlines in the README.
- Chrome Web Store screenshots must be 1280×800 or 640×400 PNG/JPEG.
